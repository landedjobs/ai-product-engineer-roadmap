## What is an AI Product Engineer?

Someone who **owns product outcomes and builds the AI features themselves** — the merge of PM judgment, full-stack ability, and LLM fluency. You decide what to build *and* ship it.

---

## The roadmap

### Stage 0 — Foundations
- Programming: Python **and** TypeScript/JavaScript.
- Web basics: a frontend framework (React/Next.js), REST APIs, a database.
- Git, deployment (Vercel/Render), reading docs fast.

### Stage 1 — Product fundamentals
- Product sense: user problems, scoping MVPs, prioritization.
- Analytics: define and read product metrics.
- Design sense: enough to ship clean UX.

### Stage 2 — LLM fluency
- Prompting well; structured outputs + validation.
- **RAG**: embeddings, vector DBs, retrieval, grounding, citations.
- **Agents**: tool use, function calling, guardrails.
- **Evals**: how to measure an AI feature's quality.
- Cost & latency: what drives them, how to manage.

### Stage 3 — Ship real AI features
- Build 2–3 end-to-end AI micro-apps (auth → UI → LLM → deploy).
- Get **real users**; iterate from feedback.
- Measure with evals; write up trade-offs.
- See [projects-to-land-an-ai-job](https://github.com/landedjobs/projects-to-land-an-ai-job).

### Stage 4 — Interview & land
- Prep AI product judgment + technical fluency + classic PM rounds — see [ai-pm-interview-prep](https://github.com/landedjobs/ai-pm-interview-prep).
- Build a portfolio + public proof (X/LinkedIn).
- **Get referred** rather than applying cold.
- Run company-specific mock interviews on [Landed](https://landed.b100x.ai).
